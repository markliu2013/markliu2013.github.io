Project Experience

Mobile Message Pushing Service

Creator

Mar 2017 – Oct 2017(3 mos)

Technologies: Golang, TCP, HTTP

The main function of the service is pull message from our system, then push to our customer's mobile device, there are broadcast message and customer message. The service can support different products in the company, and each module has their own running thread, it is very independent. The whole service is very easy to be extended, like add new products and new features.

I built the service from scratch using golang, I separated the service to different modules, include pull, pool, store, connect, push and log.

The pull module is a thread to pull messages from other system by RESTful API. It is a timer ticker to pull every 5 minutes, you can change the timer by a config file. The pull module support multiple product, each product has its own thread. It is very easy to add new API as data input, and won't affect other modules. Different API data will be converted to uniformed Message struct in service.

The pool module is a pool to store the message data from pull module, the pull module will invoke the push method of pool, it doesn't care how you store the data. There are two different type pools, QueuePool and AppQueuePool. QueuePool is the pool for customer message, AppQueuePool for broadcast message, but they are uniformed in super pool.

The store is the bridge between service and MySQL database. This module manages the connections to database and provides CRUD method for token, message, push log data.

The connect maintains UDP connection between mobile device and service, it contains UDP protocol, protocol is the Packet message format between mobile device and service, like from which byte to which byte is device type, user id, uuid.

The push module hold a queue of message to push, it depends on the connect module to connect mobile.

The log module logs the running log of the service, like message statistics, how many messages pushed. how many device connected.
